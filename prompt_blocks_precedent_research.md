# 先行事例調査メモ: PromptBlocks

調査日: 2026-06-28

## 結論

LLM向けの「視覚的に作る」先行事例は大きく2系統に分かれる。

1. 開発者・業務設計者向けのビジュアルワークフロー
2. 一般ユーザー向けのプロンプトテンプレート、最適化、保存ツール

PromptBlocksはその中間を狙う。テンプレートを選ぶだけでは硬すぎるが、ノードをつなぐワークフローは重すぎるライトユーザーに向けて、目的、条件、形式、トーンを「ブロック」として積む体験にする。

## 参考にした先行事例

| 事例 | 主な特徴 | PromptBlocksへの示唆 |
|---|---|---|
| Flowise | ドラッグ&ドロップでLLMフローを組む開発者向けツール | 視覚化は有効。ただしライトユーザーにはノード/フロー概念が重い |
| Langflow | エージェント、RAG、APIなどを視覚的に組み合わせるAIビルダー | ブロック化の対象は処理ではなく「依頼の考え方」に絞る |
| Dify | LLMアプリやワークフローを作るプラットフォーム | 業務アプリ化より、個人の1回の依頼改善に集中する余地がある |
| ChainForge | プロンプトやモデル出力を比較評価するツール | 良いプロンプトの比較学習は将来機能として相性がよい |
| AIPRM | ChatGPT向けのプロンプトテンプレート探索・共有 | テンプレートは入口として強いが、ユーザーごとの文脈編集が重要 |
| PromptPerfect | プロンプト最適化・改善 | 自動最適化より先に、ユーザーが納得して組み替えられるUIが差別化になる |

## 要件に反映したこと

- 自由なキャンバスではなく、iPhoneで使いやすい縦スクロール構成にする。
- テンプレートは最初の一歩として用意し、完成形を押し付けない。
- ブロックは「目的」「役割」「相手」「入力」「条件」「形式」「トーン」「確認」に分類する。
- 右側プレビューではなく、スマホでは下部の完成プロンプトとしてリアルタイム表示する。
- ライトユーザーの不安を減らすため、準備度スコアと不足ヒントを表示する。
